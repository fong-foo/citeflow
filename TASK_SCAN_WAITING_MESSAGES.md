## 背景

### 问题
当前初步体检扫描域名时（调用 /api/profile），用户干等没有任何反馈，可能会觉得"是不是卡住了"。

### 解决方案
超过10秒和15秒分别在对话框发消息，告诉用户在做什么、快好了。

---

## 任务1: 在 scan-chat.tsx 中添加定时消息

### 需要改的文件
`frontend/components/scan-chat.tsx`

### 实现要求

在调用 /api/profile 的地方，添加两个定时器：

```tsx
// 在调用 /api/profile 之前
const timer10s = setTimeout(() => {
  addMessage({
    sender: "probe",
    content: "正在分析你的官网内容...\n\n我们在从你的官网中提取关键信息，用来判断你的品牌属于什么行业、面向什么市场。",
    type: "text",
  });
}, 10000);

const timer15s = setTimeout(() => {
  addMessage({
    sender: "probe",
    content: "快好了，正在用 AI 推断你的行业和目标市场...\n\n马上就能看到：你的品牌属于什么行业、面向什么市场、核心产品是什么。",
    type: "text",
  });
}, 15000);

// 调用 /api/profile
const res = await fetch(`${API_BASE}/api/profile`, { ... });

// 清除定时器
clearTimeout(timer10s);
clearTimeout(timer15s);
```

### 消息内容

**10秒消息：**
```
正在分析你的官网内容...

我们在从你的官网中提取关键信息，用来判断你的品牌属于什么行业、面向什么市场。
```

**15秒消息：**
```
快好了，正在用 AI 推断你的行业和目标市场...

马上就能看到：你的品牌属于什么行业、面向什么市场、核心产品是什么。
```

## 验证方法

**测试1: 正常域名（< 10秒）**
1. 输入 ugreen.com
2. 如果在10秒内完成，不应该发任何额外消息

**测试2: 慢域名（10-15秒）**
1. 输入一个较慢的域名
2. 超过10秒应该发第一条消息
3. 在15秒前完成不应该发第二条消息

**测试3: 很慢域名（> 15秒）**
1. 输入一个很慢的域名
2. 超过10秒发第一条消息
3. 超过15秒发第二条消息
4. 完成后两条消息都应该在对话框中

## 注意事项

1. **定时器要清除** — 完成后必须 clearTimeout，避免重复发消息
2. **消息类型用 text** — 不是 profile_card 或 confirm_buttons
3. **消息内容要友好** — 不要太技术化，用用户能理解的语言
4. **不要阻塞主流程** — 定时器是异步的，不影响 /api/profile 调用
